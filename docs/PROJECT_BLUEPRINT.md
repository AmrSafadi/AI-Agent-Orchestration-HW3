# Project Blueprint

This document is the single source of truth for the project vision. It is written for three readers:

- a course grader who needs to evaluate alignment with the assignment,
- a teammate who needs to understand the system before coding,
- another AI assistant that may continue implementation later.

## 1. Project Motivation

The project demonstrates how a broad generative task can be decomposed into an organized multi-agent workflow. Instead of asking one model to "write a book and make a PDF", the system separates reasoning-heavy work from deterministic engineering work.

The intended final product is a professional article/book generated by a CrewAI team and rendered as a LaTeX PDF. The project also demonstrates production-minded ideas from the course: clear agent roles, task context, sequential orchestration, validation, observability, cost awareness, and reliable artifact generation.

## 2. Assignment Requirements

The final project must produce a professional PDF that includes:

- Multiple specialized agents
- CrewAI orchestration
- Professional PDF generation
- Cover page
- Table of contents
- Chapters and sections
- Bibliography and citations
- Images
- Python-generated graph
- Mathematical formula
- Table
- Hebrew-English mixed text section
- Final PDF output

Version 1 uses a sequential CrewAI process and exactly five agents:

- Planner Agent
- Research Agent
- Writer Agent
- Reviewer Agent
- LaTeX Agent

Citation management, graph generation, validation, and PDF compilation remain deterministic Python components.

## 4. Final System Architecture

The final system has two major layers:

1. CrewAI reasoning layer
2. Deterministic Python harness layer

```text
Project Config
  -> Sequential CrewAI Crew
     -> Planner Agent
     -> Research Agent
     -> Writer Agent
     -> Reviewer Agent
     -> LaTeX Agent
  -> Structured Artifacts
  -> Deterministic Harness
     -> CitationManager
     -> GraphGenerator
     -> Validator
     -> LaTeX Renderer
     -> PDFCompiler
  -> Final PDF
```

## 5. Agent Responsibilities

### Planner Agent

Creates the document blueprint: title, audience, chapter plan, section plan, required feature placement, page estimate, and acceptance checklist.

### Research Agent

Produces a research pack from the plan: key concepts, terminology, source candidates, chapter notes, and unsupported-claim warnings.

### Writer Agent

Writes the manuscript from the plan and research context. It includes placeholders for citations, image, graph, table, formula, and Hebrew-English text.

### Reviewer Agent

Reviews the manuscript for clarity, consistency, assignment coverage, and course alignment. It produces a review report and corrected manuscript.

### LaTeX Agent

Creates a LaTeX assembly specification. It does not compile the PDF. It prepares rendering intent for deterministic Python code.

## 7. Data Flow

```text
config/
  -> generated/intermediate/book_plan.json
  -> generated/intermediate/research_pack.json
  -> generated/intermediate/manuscript.md
  -> generated/intermediate/review_report.json
  -> generated/intermediate/latex_spec.json
  -> references.bib
  -> agent_pipeline_graph.png
  -> validation_report.json
  -> generated LaTeX files
  -> final.pdf
```

Committed example files use the `sample_` prefix under `data/intermediate/`. Dry-run copies those examples into `generated/intermediate/` and never calls `crew.kickoff()`. Future real CrewAI outputs will use the same generated runtime directory.

- Keep deterministic components deterministic.
