# ISO/IEC 25010 Quality Mapping

Per submission guideline 13, this document maps the eight ISO/IEC 25010 product
quality characteristics to concrete evidence in this project.

| # | Characteristic | How it is addressed |
|---|---|---|
| 1 | **Functional suitability** | Produces the required document with every assignment feature (cover, TOC, chapters, image, Python graph, table, formula, Hebrew-English BiDi, bibliography). The delivered document is **primarily Hebrew (RTL)** with English inline only for technical terms, and the Hebrew-English requirement is met via an embedded English LTR BiDi block (`templates/latex/chapter.tex.j2`); `config/latex.json` is now Hebrew-primary. `document/validators.py` checks completeness; the renderer assembles them; tests assert correctness. |
| 2 | **Performance efficiency** | Deterministic harness avoids unnecessary model calls; the **dry-run default** runs the full pipeline with zero API cost; the `ApiGatekeeper` bounds provider usage via rate limits, retries, and backpressure. |
| 3 | **Compatibility** | `uv`-managed, runs on Windows (PowerShell) and POSIX shells; uses standard, portable formats (JSON config, Markdown, LaTeX, BibTeX); CrewAI import is optional with a dry-run fallback. |
| 4 | **Usability** | A clear CLI (`--dry-run` / `--run-crew` / `--build-pdf`) with explicit status output; `docs/USAGE.md` documents the experience and Nielsen heuristics; safe defaults prevent accidental cost. |
| 5 | **Reliability** | Dry-run safety, graceful degradation (the compiler reports cleanly when no TeX toolchain is present), gatekeeper retries on transient failures, and a test suite of **109 tests passing, 2 skipped, at 91.96% coverage**, enforced by an **85% gate** (pyproject `fail_under=85`). |
| 6 | **Security** | No secrets in source — environment variables only, `.env-example` with dummy values, `.gitignore` for `.env`/`*.key`/`*.pem`; all provider calls funnel through one monitored gatekeeper; agents run with `allow_delegation=False` (least privilege). |
| 7 | **Maintainability** | Modular `src/bookgen/{shared,document,harness,orchestration,latex,sdk,research}` packages; an **SDK facade** as the single entry point; small single-responsibility modules (every code file ≤150 lines); Ruff lint + format, pre-commit hook, and CI; Pydantic contracts and DRY shared helpers. |
| 8 | **Portability** | Package-relative and configurable paths (no hardcoded machine paths); `uv` + `pyproject.toml` + `uv.lock` for reproducible installs; all tunables live in versioned `config/*.json`. |

## Notes

- Runtime configuration-version validation (`shared/config.py`) enforces the
  expected config version, supporting reliability and maintainability.
- Logging is configured from `config/logging_config.json` (`dictConfig`),
  supporting observability and maintainability.
